# IT Helpdesk Ticket System Upgrade — Project Plan

A self-directed project management portfolio piece simulating the planning, execution, and rollout of a helpdesk ticketing system upgrade — built to demonstrate proficiency with real-world PM tools and practices (Jira, Gantt timelines, stakeholder mapping, and project documentation).

> **Note:** This is a simulated project created to showcase project management skills and tooling. All stakeholders, timelines, and metrics are illustrative rather than drawn from an actual deployment.

---

## Project Summary

This project covers selecting and configuring a new ticketing platform, migrating open and historical tickets from the legacy system, and configuring automated SLA rules with escalation alerts. It also includes setting up staff-facing UI/mobile access, training helpdesk staff, and running a 2-week parallel-run period before full cutover. The goal is faster ticket resolution times and minimal disruption for stakeholders during the transition.

## The Problem

The organization's legacy ticketing system lacks automated SLA tracking and mobile access, leading to slower resolution times, inconsistent prioritization, and a growing backlog as ticket volume increases. Full details are in the [Project Charter](docs/project-charter.pdf).

## Approach

The project was planned and tracked using a Kanban workflow across four phases:

| Phase | Focus |
|---|---|
| **Planning** | Requirements gathering, platform selection |
| **Development** | Platform configuration, SLA rules, routing logic |
| **Testing** | Test case execution, user acceptance testing |
| **Training & Rollout** | Staff training, parallel-run, cutover |

## Tools Used

- **Jira** — Kanban board, epics, tasks, dependencies
- **Gantt timeline tool** — 6-week schedule with task dependencies
- **draw.io** — stakeholder and communication-flow diagram
