# Communication Plan: IT Helpdesk Ticket System Upgrade

**Project Lead:** Toni Lewis
**Last Updated:** July 2026

---

## Purpose

This plan defines how project updates, decisions, and issues will be communicated across stakeholders throughout the 6-week rollout, ensuring everyone has the right information at the right time without unnecessary noise.

## Communication Matrix

| From | To | Frequency | Method | Content |
|---|---|---|---|---|
| Project Manager | Product Owner | Weekly | Status report (email/doc) | Overall progress, blockers, upcoming milestones, budget/timeline flags |
| Project Manager | Backend Team | Daily | Standup (15 min) | Task progress, blockers, same-day priorities |
| Project Manager | Frontend Team | Daily | Standup (15 min) | Task progress, blockers, same-day priorities |
| Project Manager | UX | Daily | Standup (15 min) | Task progress, blockers, same-day priorities |
| Backend Team | End Users | As needed (Weeks 5-6) | Email/announcement | Ticket resolution status during parallel-run and cutover |
| Project Manager | All teams | Weekly | Team sync (30 min) | Cross-team dependencies, upcoming phase transitions |
| Vendor Support | Project Manager | As needed | Call/email | Platform configuration support, troubleshooting |

## Escalation Path

If a blocker cannot be resolved within the team:
1. Raised at daily standup by task owner
2. If unresolved in 24 hours, escalated to Project Manager directly
3. If it affects timeline or budget, Project Manager raises to Product Owner within 48 hours

## Key Milestone Communications

| Milestone | Audience | Method |
|---|---|---|
| Platform selected (end of Week 1) | Product Owner, all teams | Email summary |
| Development complete (end of Week 3) | Product Owner | Status report |
| UAT sign-off (end of Week 5) | Product Owner, helpdesk staff | Email + sign-off document |
| Parallel-run start (Week 5) | End users, helpdesk staff | Org-wide announcement |
| Full cutover (Week 6) | End users, all stakeholders | Org-wide announcement |

## Notes

All weekly status reports and sync notes are stored in the project's shared documentation space for reference by any stakeholder at any time.
